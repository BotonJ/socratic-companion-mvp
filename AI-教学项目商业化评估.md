### 二、 客户端（iOS/Web）如何接入大模型服务器？
无论你开发的是 Web 还是 iOS，**绝对不能**将大模型（如 Claude 或 OpenAI）的 API Key 直接写在前端代码里，这会导致秘钥泄露和恶意刷量。标准的接入架构如下：

### 🤖 Assistant

浏览器插件由于浏览器的安全沙箱机制，**无法直接在系统中执行**终端里的 `claude code` 或 `openclaw` 命令。要实现通信，必须在两者之间建立一个**本地桥接服务（Local Web Server）**。
整个通信与执行的架构流程如下：
**1. 浏览器插件（数据抓取与发送）**

- 插件使用 JavaScript（Content Script）读取当前网页的 DOM 树，提取核心文章区域的纯文本，剔除广告、导航等无关内容。
- 插件将提取的文本与你的“虚拟导师设定 Prompt”进行拼接。
- 插件向本地地址（例如 `http://127.0.0.1:8080/ask`）发起 HTTP POST 请求，发送拼接好的内容。
**2. 本地桥接服务（请求接收与命令转换）**

- 你在本地电脑上运行一个轻量级的后端服务来监听该端口。用 JavaScript (Node.js 环境) 或 Java 都可以非常快速地写出这种简单的 HTTP 接口。
- 这个本地服务收到插件的请求后，负责在后台通过**子进程（Child Process）**静默调用终端命令。
- 例如，在 Node.js 中使用 `exec` 或 `spawn` 方法，执行类似 `openclaw --prompt "你的组合文本"` 的系统命令。
**3. 终端工具执行与数据捕获**

- 系统在后台运行 `claude code` 或 `openclaw` 并消耗 Token 与大模型通信。
- 本地桥接服务实时捕获这些终端工具的标准输出流（stdout）。
**4. 结果返回与侧边栏渲染**

- 本地服务将捕获到的回复，通过 Server-Sent Events (SSE) 以流式传输的方式返回给浏览器插件。
- 插件侧边栏接收到数据流，以“打字机”效果实时渲染 Markdown 文本，完成伴读交互。
**补充的进阶路径（Native Messaging）：**
如果你不想在本地挂着一个 HTTP 服务器，Chrome/Edge 浏览器底层支持一种叫 **Native Messaging** 的 API。它允许浏览器插件通过标准输入/输出（stdin/stdout）直接与本地的 Python 或 Node.js 脚本通信。配置好特定的 JSON 清单文件后，插件就能直接唤起脚本，脚本再去调用 OpenClaw，链路更短，但初期的系统权限和文件配置相对繁琐。
整体来看，`插件 -> 本地 HTTP 服务 -> CLI 工具` 是最易于调试和跑通 MVP 的方案，并且全程数据都在本地机器上流转，隐私安全性极高。

1. **冷启动与内存开销：** 如果没有配置并开启常驻内存的服务器模式（`qmd mcp`），每一次互动提问，系统都要把几百兆的模型重新载入内存、计算、再释放。这种剧烈的延迟会彻底破坏苏格拉底式教学的互动“心流”。
2. **垂直领域的语义盲区：** 这种轻量级的本地模型在处理日常对话时还能应付，但当你用它来解析维特根斯坦的哲学概念、或者庞大的人工智能技术发展史时，它在多语言和复杂学术概念上的语义聚类能力非常羸弱，导致根本搜不到准确的内容。

### 3. 结构化逻辑网：GraphRAG（知识图谱驱动）

- **如何形成认知**：当你在阅读原著并询问某个复杂概念时，系统检索到的不再是包含这个词的零散段落，而是这个概念在全书中的逻辑树。它能告诉用户：“这个概念由 A 推导而来，并在后文影响了 B 决策”。
- **优势**：这是真正意义上的认知解构。对于高净值的学术研究、复杂资料审计，知识图谱能提供极其震撼的“全局视野”。

### 2. 本地 Web UI (Streamlit / Gradio) —— AI 产品的“快手”方案
这是目前 AI 初创项目最常用的套壳方式。

- **实现逻辑**：你的 CLI 程序在后台运行，前端使用 Streamlit 搭建一个简单的网页。
- **优势**：
  - **极速开发**：你只需要用 Python 写几行 UI 代码，就能拥有按钮、输入框和 Markdown 渲染。

  - **数据隔离**：依然保持本地运行，不涉及云端隐私问题。

### 3. Tauri —— 现代化的桌面客户端（推荐）
如果你希望真正交付一个 `.app` (Mac) 或 `.exe` (Windows) 文件，**Tauri 是目前优于 Electron 的选择**。

- **实现逻辑**：后端使用 Rust（负责调用系统 CLI 和处理文件），前端使用任何你熟悉的 Web 技术（React/Vue）。
- **优势**：
  - **极其轻量**：打包出来的安装包可能只有几 MB，而 Electron 通常要上百 MB。
  - **系统原生感**：它调用系统自带的 WebView，运行速度极快，适合 Mac mini 这种追求性能的设备。
  - **无缝集成**：它可以直接把你的 CLI 程序打包在安装包内部，用户双击打开就是一个漂亮的图形界面，完全感知不到背后的命令行。

- **效果**：它能自动读取你的命令行参数（如 `--persona`, `--book`），并瞬间生成一个简单的 Windows/Mac 配置窗口。
- **适用场景**：仅用于任务开始前的“配置”，不适合长期的对话交互。
